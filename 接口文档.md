## 基础说明

### 认证方式
所有需要认证的接口需要在请求头中携带 JWT Token：
```
Authorization: Bearer <your_access_token>
```


### 统一响应格式
```typescript
interface Result<T> {
  code: number;      // 状态码
  message: string;   // 消息
  data: T;           // 数据
}
```


### 状态码说明
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未认证/Token无效 |
| 403 | 无权限 |
| 500 | 服务器错误 |
| 7777 | 账户被冻结 |
| 8888 | 密码已修改/强制下线 |

### 分页响应结构
```typescript
interface Page<T> {
  records: T[];        // 数据列表
  total: number;       // 总记录数
  size: number;        // 每页大小
  current: number;     // 当前页码
  pages: number;       // 总页数
}
```

## 1. 🔐 认证中心

### 1.1 用户注册

**接口地址**: `POST /users/register`  
**是否需要认证**: ❌ 否  
**权限要求**: 无

**请求参数 (Body - JSON)**:
```typescript
{
  username: string;    // 用户名，必填，2-20字符，支持字母/数字/下划线/中文
  password: string;    // 密码，必填，6-32字符
  email?: string;      // 邮箱，可选，需符合邮箱格式
  phone?: string;      // 手机号，可选，需符合中国大陆手机号格式
  organizationId?: number;  // 所属单位ID，可选
}
```


**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1234567890,
    "username": "张三",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "status": 0,
    "roleIds": [1001],
    "organizationId": null
  }
}
```


**影响**:
- 插入 `sys_user` 表
- 插入 `sys_user_role` 表（默认分配角色 1001）

### 1.2 用户登录

**接口地址**: `POST /users/login`  
**是否需要认证**: ❌ 否  
**权限要求**: 无

**请求参数 (Body - JSON)**:
```typescript
{
  userName: string;    // 用户名，必填，2-20字符（注意：字段名为 userName，非 username）
  password: string;    // 密码，必填，6-32字符
}
```


**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "username": "张三",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "status": 0,
    "roleIds": [1001, 1002],
    "roleNames": ["普通用户", "超级管理员"],
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 1800,
    "refreshExpiresIn": 604800
  }
}
```


**字段说明**:
- `token`: Access Token，有效期 30 分钟（1800秒）
- `refreshToken`: Refresh Token，有效期 7 天（604800秒）
- `expiresIn`: Access Token 剩余有效时间（秒）
- `refreshExpiresIn`: Refresh Token 剩余有效时间（秒）

**影响**: Redis 存储 Refresh Token

## ⚠️ 注意事项

### 1. Token刷新机制
- **Access Token**: 有效期30分钟
- **Refresh Token**: 有效期7天
- 当Access Token剩余时间不足一半时，响应头会返回新的Token：
  ```

Authorization-New: Bearer <new_access_token>
  ```
- 前端应检测此Header并更新本地存储的Token

### 2. 特殊状态码
- **7777**: 账号已被冻结
- **8888**: 密码已修改或已被强制下线，需要重新登录

### 3. 日期格式
所有日期字段统一使用格式：`yyyy-MM-dd HH:mm:ss`（北京时间 GMT+8）

### 4. 分页参数
- `pageNum`: 从1开始
- `pageSize`: 范围1-100，建议默认10

### 5. 权限控制
- 所有需要认证的接口必须在Header中携带Token
- 部分接口需要特定权限编码，由后端通过`@HasPermission`注解控制
- 前端可根据用户权限动态显示/隐藏功能按钮

### 6. 多租户隔离
- 普通用户只能访问本单位数据
- 超级管理员可跨租户访问所有数据
